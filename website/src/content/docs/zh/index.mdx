## 性能实测

我们在一个[真实数据库系统任务](https://github.com/datacurve-ai/deep-swe/tree/main/tasks/pebble-durability-wait-apis)上对比了 Chord v0.6.3 和 Codex-CLI v0.136.0：为 Pebble 实现 durability wait API。该任务并非简单的增删改查——需要理解 commit/WAL sync 和并发语义，跨越写入路径、事件监听、DB 生命周期等多个子系统进行推理，最终生成 8 个文件、853 行代码的补丁。

两者均使用 GPT-5.5（xhigh）：

| 指标 | Chord v0.6.3 | Codex-CLI v0.136.0 | 改进 |
|------|--------------|---------------------|------|
| **时间** | 48 分钟 | 63 分钟 | **快 24%** |
| **API 调用** | 93 次 | 118 次 | **少 21%** |
| **输入 tokens** | 6.86M | 18.47M | **少 63%** |
| **输出 tokens** | 25K | 74K | **少 66%** |
| **缓存读取 tokens** | 6.55M | 17.64M | **少 63%** |
| **成本** | **$5.58** | **$15.15** | **省 63%** |


## 为什么选 Chord

<CardGrid>
  <Card title="稳定长期运行" icon="document">
    Chord 会在长对话接近模型 token 上限前压缩早期内容，同时保留继续工作所需的信息。
  </Card>
  <Card title="网络状态全程可见" icon="rocket">
    模型调用过程中精确显示请求状态与已等待时间，再也不用猜「是不是卡住了」。
  </Card>
  <Card title="键盘优先、Vim 风格" icon="seti:vim">
    Insert / Normal 模式、消息搜索、完整 Vim 风格导航、模式切换自动切输入法。退出需连按两次。
  </Card>
  <Card title="模型组合热切换" icon="random">
    将模型分到可复用的池（`fast`、`thinking`、`cheap`），运行时用 `/models` 或 `Ctrl+P` 切换。运行时按池内顺序自动 fallback。
  </Card>
  <Card title="资源占用极低" icon="seti:powershell">
    低内存、低 CPU。macOS 上电源感知：工作时阻止系统休眠，空闲后自动放行。
  </Card>
  <Card title="可远程操控" icon="external">
    `chord headless` 提供 stdio JSON 控制面，配合 [chord-gateway](https://github.com/keakon/chord-gateway) 可从任意聊天平台操控，离开桌面也能在手机上用。
  </Card>
  <Card title="一键导入外部会话" icon="import">
    一条命令即可从 Claude Code、Codex、OpenCode 迁移。对话记录和工具调用自动转换，无缝恢复原有上下文。
  </Card>
</CardGrid>

## 三步上手

<CardGrid>
  <Card title="1. 安装" icon="seti:go">
    已安装 Go 1.26.3+ 时：

    ```bash
    go install github.com/keakon/chord/cmd/chord@latest
    ```

    未安装 Go 1.26.3+ 时，可从 [GitHub Releases](https://github.com/keakon/chord/releases) 下载预构建二进制，解压后把 `chord` 放到 `PATH` 中。
  </Card>
  <Card title="2. 先运行一次初始化向导" icon="seti:lock">
    在交互式终端里直接运行：

    ```bash
    chord
    ```

    如果缺少 `config.yaml`，Chord 会启动一次性的初始化向导。向导会创建最小可用的 `config.yaml`，必要时再创建 `auth.yaml`。

    如果你更希望手写 YAML，或需要不同的 provider / 模型配置，见[快速开始](/chord/zh/quickstart/)。
  </Card>
  <Card title="3. 在项目里启动" icon="rocket">
    ```bash
    cd my-project
    chord
    ```
    在某个项目中首次运行时，Chord 会按需创建 `.chord/`。
  </Card>
</CardGrid>
