## 三步上手

<CardGrid>
  <Card title="1. 安装" icon="seti:go">
    下载预构建二进制不需要安装 Go。已安装 Go 1.26.3+ 时，也可以直接用源码安装：

    ```bash
    go install github.com/keakon/chord/cmd/chord@latest
    ```

    未安装 Go 1.26.3+ 时，可从 [GitHub Releases](https://github.com/keakon/chord/releases) 下载预构建二进制，解压后把 `chord` 放到 `PATH` 中。
  </Card>
  <Card title="2. 先运行一次初始化向导" icon="seti:lock">
    在交互式终端里直接运行：

    ```bash
    chord
    ```

    如果缺少 `config.yaml`，Chord 会启动一次性的初始化向导。向导会创建最小可用的 `config.yaml`，必要时再创建 `auth.yaml`。

    如果你更希望手写 YAML，或需要不同的 provider / 模型配置，见[快速开始](/chord/zh/quickstart/)。
  </Card>
  <Card title="3. 在项目里启动" icon="rocket">
    ```bash
    cd my-project
    chord
    ```
    在某个项目中首次运行时，Chord 会按需创建 `.chord/`。
  </Card>
</CardGrid>

## 为什么选 Chord

<CardGrid>
  <Card title="长会话更省上下文" icon="seti:clock">
    请求时剪裁陈旧工具输出，并用类型化摘要保留大块结果的关键信息；接近上限前再压缩早期内容。
  </Card>
  <Card title="轻量常驻" icon="approve-check-circle">
    近乎瞬间加载上千条消息的历史会话，退出无需等待；内存占用低，空闲数分钟后卸载 LSP/MCP 资源。
  </Card>
  <Card title="网络状态全程可见" icon="information">
    模型调用过程中精确显示请求状态与已等待时间，再也不用猜「是不是卡住了」。
  </Card>
  <Card title="模型组合热切换" icon="random">
    将模型分到可复用的池（`fast`、`thinking`、`cheap`），运行时用 `/models` 或 `Ctrl+P` 切换。运行时按池内顺序自动 fallback。
  </Card>
</CardGrid>

## 性能实测

这些数据是特定场景下的依据，不是对所有任务的通用承诺。它们主要展示 Chord 的上下文剪裁、低开销 TUI 和可预测退出行为在哪些地方有价值。

### 真实编码任务

我们在一个[真实数据库系统任务](https://github.com/datacurve-ai/deep-swe/tree/main/tasks/pebble-durability-wait-apis)上对比了 Chord v0.6.3 和 Codex-CLI v0.136.0：为 Pebble 实现 durability wait API。该任务并非简单的增删改查——需要理解 commit/WAL sync 和并发语义，跨越写入路径、事件监听、DB 生命周期等多个子系统推理。

| 指标 | Chord v0.6.3 | Codex-CLI v0.136.0 | 改进 |
|------|--------------|---------------------|------|
| **耗时** | **46m21s** | 61m18s | **快 24%** |
| **模型调用** | **93 次** | 118 次 | **少 21%** |
| **输入 tokens** | **6.86M** | 18.47M | **少 63%** |
| **输出 tokens** | **25K** | 74K | **少 66%** |
| **缓存读取 tokens** | **6.55M** | 17.64M | **少 63%** |
| **成本** | **$5.58** | $15.15 | **省 63%** |

注：
- 两次运行均使用 GPT-5.5（xhigh）。
- 耗时排除环境准备和最终收尾时间，包含模型交互、代码修改和测试执行。
- 该任务参考答案规模为 8 个文件、670 行变更；实际模型输出可能因单元测试、注释或实现路径不同而更大或更小。

### 应用启动与内存

我们还测了应用交互层：从启动到可以输入、正常退出耗时，以及空会话和加载 200 条消息后的内存。

| 应用 | 启动到可输入 | 正常退出 | 空会话内存 | 200 消息内存 |
|------|--------------|----------|------------|--------------|
| Chord v0.6.3 | **&lt;1s** | **&lt;1s** | **31.6MB** | **~40MB** |
| Codex-CLI v0.136.0 | **&lt;1s** | ~20s | 35.8MB | ~80MB |
| Claude Code v2.1.163 | 32s | ~2s | 156.3MB | &gt;300MB |

注：
- Codex-CLI 退出时会等待收尾，约 20 秒超时后退出。
- Claude Code 启动时会等待，约 30 秒超时后才可以正常输入。
- 不同会话和环境的内存占用会有差异，表中数据仅用于估算。
