## 为什么选 Chord

<CardGrid>
  <Card title="长会话不崩" icon="document">
    自动 compaction 把早期对话压成上下文摘要，会话超出 context window 仍然能继续——保留了继续工作所需的信息。
  </Card>
  <Card title="网络状态全程可见" icon="rocket">
    模型调用过程中精确显示请求状态与已等待时间，再也不用猜"是不是卡住了"。
  </Card>
  <Card title="键盘优先、Vim 风格" icon="seti:vim">
    Insert / Normal 模式、消息搜索、完整 Vim 风格导航、模式切换自动切输入法。退出需连按两次。
  </Card>
  <Card title="模型组合热切换" icon="random">
    把模型分到可复用的池（`fast`、`thinking`、`cheap`），运行时用 `/models` 或 `Ctrl+P` 切换。运行时按池内顺序自动 fallback。
  </Card>
  <Card title="小 VPS 上能跑很久" icon="seti:powershell">
    低内存、低 CPU。macOS 上电源感知：工作时阻止系统休眠，空闲后自动放行。
  </Card>
  <Card title="可远程操控" icon="external">
    `chord headless` 提供 stdio JSON 控制面，配合 [chord-gateway](https://github.com/keakon/chord-gateway) 可从任意聊天平台操控。
  </Card>
</CardGrid>

## 三步上手

<CardGrid>
  <Card title="1. 安装" icon="seti:go">
    ```bash
    go install github.com/keakon/chord/cmd/chord@latest
    ```
    需要 Go 1.26+。
  </Card>
  <Card title="2. 配置凭据" icon="seti:lock">
    ```bash
    mkdir -p ~/.config/chord && chmod 700 ~/.config/chord
    echo 'anthropic:\n  - "$ANTHROPIC_API_KEY"' \
      > ~/.config/chord/auth.yaml
    ```
    其他 provider 见 [快速开始](/chord/zh/quickstart/)。
  </Card>
  <Card title="3. 在项目里启动" icon="rocket">
    ```bash
    cd my-project
    chord
    ```
    首次启动会按需创建项目级 `.chord/`。
  </Card>
</CardGrid>
