## Why Chord

<CardGrid>
  <Card title="Stable for the long haul" icon="document">
    Chord can summarize earlier turns before a long conversation gets too large
    for the model, while preserving the information needed to continue.
  </Card>
  <Card title="You see the network state" icon="rocket">
    Precise request status and elapsed wait time during model calls. Never
    wonder if it's stuck again.
  </Card>
  <Card title="Keyboard-first, Vim-style" icon="seti:vim">
    Insert / Normal modes, message search, full Vim-flavoured navigation,
    automatic input-method switching. Quitting takes two taps.
  </Card>
  <Card title="Hot-swap model setups" icon="random">
    Group models into reusable pools (`fast`, `thinking`, `cheap`) and switch
    them at runtime via `/models` or `Ctrl+P`. The runtime falls back through
    the ordered list automatically.
  </Card>
  <Card title="Extremely lightweight" icon="seti:powershell">
    Low memory and CPU footprint. macOS power-aware: prevents idle sleep
    while work is active, lets the system sleep again when idle.
  </Card>
  <Card title="Drive it remotely" icon="external">
    `chord headless` exposes a stdio JSON control plane; pair with
    [chord-gateway](https://github.com/keakon/chord-gateway) to operate Chord
    from any chat surface — even from your phone when you are away.
  </Card>
  <Card title="Import existing sessions" icon="import">
    Migrate from Claude Code, Codex, or OpenCode with a single command.
    Import turns and tool history are converted automatically — resume
    where you left off without losing context.
  </Card>
</CardGrid>

## Three-step setup

<CardGrid>
  <Card title="1. Install" icon="seti:go">
    If you already have Go 1.26+ installed:

    ```bash
    go install github.com/keakon/chord/cmd/chord@latest
    ```

    If not, download a prebuilt binary from [GitHub Releases](https://github.com/keakon/chord/releases), extract it, and put `chord` on your `PATH`.
  </Card>
  <Card title="2. Configure model and credentials" icon="seti:lock">
    ```bash
    mkdir -p ~/.config/chord && chmod 700 ~/.config/chord
    cat > ~/.config/chord/config.yaml <<'YAML'
    providers:
      openrouter:
        type: chat-completions
        api_url: https://openrouter.ai/api/v1/chat/completions
        models:
          openai/gpt-5.5:
            limit:
              context: 400000
              input: 272000
              output: 128000
            modalities:
              input: [text, image]
    model_pools:
      default:
        - openrouter/openai/gpt-5.5
    YAML
    cat > ~/.config/chord/auth.yaml <<'YAML'
    openrouter:
      - "$OPENROUTER_API_KEY"
    YAML
    ```
    Other OpenRouter models and provider setup: see [Quickstart](/chord/quickstart/).
  </Card>
  <Card title="3. Run from your project" icon="rocket">
    ```bash
    cd my-project
    chord
    ```
    On first run, Chord creates `.chord/` for project-level config.
  </Card>
</CardGrid>
